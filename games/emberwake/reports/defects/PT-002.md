# PT-002 — spell.mend declares menu usability but no scene can cast spells from the menu

- **Persona:** rusher
- **Seed:** 1101
- **Script:** `reports/playtest/rusher/actions.json`
- **Trace:** `reports/playtest/rusher/trace.jsonl`, lines 1357-1380 (pre-boss pause-menu
  sequence at F3 (9,8)) — the only healing paths available were items
- **Severity:** low (authored capability unreachable; no crash/stuck)
- **Owning skill:** m8-scene-smith (pause menu / status scene), with m8-story-smith
  (content intent) to confirm

## Repro
`content/spells.json`: `spell.mend` has `"usable_in": ["battle", "menu"]`. Open the
overworld pause menu (rusher trace lines 1357-1380: Items → use draughts/emberdew).
There is no path to cast Mend outside battle: `scenes/pause_menu.gd` offers only
Items / Status / services / Quit; `scenes/inventory.gd` handles items only;
`scenes/status.gd` is read-only. `Game.use_item_on_member` exists for items, but no
spell equivalent is exposed by any scene.

## Expected
Content declaring `usable_in: ["menu"]` is reachable from the menu layer (as it is for
items with the same declaration — warm_draught/emberdew were menu-usable in this run),
or the content should not declare menu usability.

## Observed
The rusher had to spend purchased consumables (draughts) for out-of-battle healing while
the party's healer had MP and a menu-tagged heal spell. Out-of-battle spell casting is
authored in content but unreachable in the engine's scene layer — orphaned capability.

## Suggested fix
Either add a Spells row to the pause menu (menu-usable spells via a member picker,
mirroring inventory's Use flow), or drop `"menu"` from `spell.mend.usable_in`.

## Disposition (m8-conductor)
Deferred to M1 (m8-menu-wright). The GDD does not require menu spellcasting; out-of-battle healing is covered by items + inn. Already queued in retrospectives/queue.md as a menu-wright skill-edit candidate (pause menu should gain a Magic row when any usable_in:menu spell exists).
